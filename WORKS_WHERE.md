# Что где работает

Быстрая памятка: какие функции доступны локально (на вашем компе через
`InstaContentEngine.pyw`), какие требуют ключей, и что работает только в
облаке (Render/Railway, см. [DEPLOY.md](DEPLOY.md)).

---

## 🟢 Локально, без единого ключа

Запустили `.pyw` — сразу доступно:

- Нативное окно приложения (Create / Trends / Calendar / Grid / Admin)
- Просмотр макетов слайдов 1080×1350, редактирование overlay-текста (Apply / Reset)
- **Calendar** и **Content pillars** (микс + «что постить сегодня»)
- **Grid** — превью ленты, переключатель 📱 Mobile / 🖥 Desktop
- **🎬 Make Reel** — рендер вертикального видео из слайдов (ffmpeg локальный) + Download MP4
- **↶ Undo** и **autosave** черновиков, эмодзи-пикер
- **⬇ Backup** — скачать ZIP базы + сгенерированных медиа
- **Save ZIP to Downloads** (экспорт поста)
- Cost-бейдж (покажет `$0.00`, пока не было генераций)

## 🟡 Локально, но нужен ключ в `backend/.env`

| Функция | Нужный ключ |
|---|---|
| Генерация подписи, **↻ Variations**, **Adapt to MLMG**, web-источники | `OPENROUTER_API_KEY` |
| Сток-картинки для слайдов | `UNSPLASH_ACCESS_KEY` и/или `PEXELS_API_KEY` |
| AI-генерация картинок | `OPENROUTER_API_KEY` (image-модель в `DEFAULT_IMAGE_MODEL`) |
| **Trend Finder** (Refresh trends) | `INSTAGRAM_ACCESS_TOKEN` + `INSTAGRAM_USER_ID` |
| **Insights** (reach / saves / shares) | `INSTAGRAM_ACCESS_TOKEN` + `INSTAGRAM_USER_ID` |
| **Hashtag Analyze** (IG-часть) | `INSTAGRAM_ACCESS_TOKEN` + `INSTAGRAM_USER_ID` |
| **Publish now** (фото / карусель) | `IMGBB_API_KEY` + IG-токен |
| **Scheduled publish, пока комп включён** | `IMGBB_API_KEY` + IG-токен |
| Cost-трекинг (реальные суммы) | считается сам при генерациях |

> Hashtag Analyze работает и **без** IG-токена — тогда ранжирование идёт только
> по эвристике из собранных данных конкурентов (без абсолютного media_count).

## 🔵 Только в облаке (cloud mode)

Ровно две вещи, каждая — по технической причине, а не «недоделке»:

| Функция | Почему нужен сервер 24/7 |
|---|---|
| **Публикация по расписанию, когда компьютер выключен** | У Instagram нет отложенной публикации на своей стороне через API — publish должен кто-то вызвать в назначенный момент. Локально это делает ваш комп (если открыт). Выключили — нужен всегда-включённый backend. |
| **Авто-публикация Reels** (видео → сразу в IG) | Instagram скачивает видео по публичному URL. imgbb хостит только картинки, поэтому MP4 отдаёт сам backend по `PUBLIC_BASE_URL` — а публичный адрес есть только у задеплоенного сервера. Локально Reel **рендерится и качается MP4**, публикацию делаете вручную. |

Развёртывание — один клик по `render.yaml` в Render, ~$7/мес (starter-план;
free-tier «засыпает» и пропустит расписание). Подробно — [DEPLOY.md](DEPLOY.md).

## Итог одной строкой

```
без ключей   →  макеты, видео, календарь, grid, pillars, backup, undo/autosave
+ ключи      →  вся генерация, тренды, инсайты, publish now, scheduled (комп включён)
только облако →  scheduled при выключенном компе  +  авто-публикация Reels
```

**~90% функциональности работает локально.** Облако добавляет по сути одно:
публикацию без вашего компьютера.
